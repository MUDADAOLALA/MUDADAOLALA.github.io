docker

把环境和代码一起装到一个容器中进行发送，以解决因为环境不同造成的不可运行，规避了环境不同导致的bug。即软件跨环境迁移的问题。
Docker架构：Clients、Hosts、Registries
Clients：客户端发送命令给守护进程
Hosts：Docker创建后会以守护进程(daemon，后台运行的进程)的形式存在。里面有image镜像（类）和container容器（对象）。
Registries：镜像来自于远程的仓库：官方、私服
配置了阿里云的镜像加速器
服务（守护进程）相关命令
通过命令操作image
docker images查看镜像
docker search redis搜索镜像
docker pull redis 下载镜像
docker rmi (image id)删除镜像
通过命令操作container（最重要）
docker run -it –name=c1 ubuntu 进入容器内部
i:保持一直运行 t:分配终端
docker ps 查看正在运行的容器
docker ps -a 查看历史容器
docker run -id –name=c2 ubuntu
d后台运行，exit不会退出，需要用命令进入
docker exec -it c2 /bin/bash进入容器内部
it交互式容器 id守护式容器
docker stop c2 关闭容器
docker start c2开启
docker rm c1删除容器

数据卷是宿主机中的一个目录或者文件，容器目录和数据卷目录绑定后，修改会同步。多对多
数据卷作用：容器数据持久化、外部机器和容器间接通信、容器间数据交换

当容器中的网络服务需要被外部机器访问时，可以将容器中提供服务的端口映射到宿主机的另一个端口上，外部机器访问这个端口从而间接访问容器的服务——端口映射

图片大小格式问题

图片容量大小调整：

1、用系统自带画图打开，点击重设大小，在保持纵横比的前提下调整像素

2、系统自带的照片打开，点击重设大小，点击最适合用于个人资料图片和缩略图

3、word中插入图片，点击选中图片后在上方菜单栏点击图片格式，然后压缩图片

4、其他软件、在线网站

5、手机上发送到wps中，进行图片压缩

6、改后缀名没用

iphone过热问题

iphone啥也没做突然好烫，自动重启

原因即对应解决方案：

1、后台软件开的过多，检查后台是否开了很多软件，比如耗内存高的视频软件、相机等，并及时清理后台

2、长时间使用需要消耗大量资源的软件，比如录像，大型游戏，看视频，及时关掉并休息片刻再使用

3、长时间开热点，关掉热点

4、在太阳下或闷热环境中使用，到凉快的环境使用

5、充电时过度使用手机，充完电再用或停止充电使用

6、ios系统出现了问题，换用可以正常使用的系统，不要着急更新，等待修复补丁

7、硬件损坏，如果没有使用情况导致，考虑硬件问题，及时送往维修处维修

提醒：睡觉时请不要把手机放在枕边，更不能在睡觉时放在身边充电。
